💻 Sécurité Web

Menaces web courantes

Internet offre de nombreuses opportunités mais aussi des risques. Voici les menaces les plus courantes.

🎣 Phishing

Le phishing est une tentative de vol d'informations par tromperie.

  • Les attaquants envoient des emails faux qui semblent authentiques
  • Ils demandent de cliquer sur des liens malveillants
  • Ou de fournir des informations sensibles
  • Comment vous protéger : Vérifiez l'adresse email, ne cliquez pas sur les liens suspects, regardez les URLs avec attention

🌐 Cross-Site Scripting (XSS)

Une attaque qui injecte du code malveillant dans une page web.

  • L'attaquant insère du JavaScript malveillant
  • Il s'exécute dans le navigateur de la victime
  • Peut voler des cookies ou des données personnelles

💉 Injection SQL

Une technique qui exploite les failles dans les applications web.

  • L'attaquant insère du code SQL dans les champs de formulaire
  • Le serveur exécute le code malveillant
  • Permet d'accéder ou de modifier les données de la base de données
  • Exemple : Dans un champ de recherche : ' OR '1'='1

🔒 Cross-Site Request Forgery (CSRF)

Une attaque qui vous fait exécuter des actions sans votre consentement.

  • L'attaquant crée une page web malveillante
  • Vous visitez cette page sans le savoir
  • Elle envoie une requête à votre compte en utilisant vos données d'authentification
  • Peut transférer de l'argent ou changer vos paramètres

🚪 Attaques par force brute

Les attaquants tentent de deviner votre mot de passe en essayant plusieurs combinaisons.

  • Utilisent des listes de mots de passe courants
  • Essaient des variations de votre nom ou date de naissance
  • Peuvent utiliser des outils automatisés
  • Protection : Mots de passe forts, limitation de tentatives, 2FA

📱 Clickjacking

Une attaque qui vous trompe pour cliquer sur quelque chose que vous ne pensez pas cliquer.

  • Un élément invisible recouvre un élément visible
  • Vous cliquez sur ce que vous voyez mais activez ce qui est caché
  • Peut vous abonner à un service ou partager du contenu

🔓 Fuites de données

L'exposition accidentelle ou intentionnelle de données sensibles.

  • Données mal configurées dans le cloud
  • Bases de données non sécurisées
  • Transmissions sans chiffrement
  • Employés négligents

🛡️ Pratiques de sécurité web

Pour les utilisateurs :

  • ✅ Vérifiez que les sites utilisent HTTPS (🔒)
  • ✅ Utilisez des mots de passe forts et uniques
  • ✅ Activez la double authentification (2FA)
  • ✅ Soyez vigilant face aux emails suspects
  • ✅ Ne cliquez pas sur les liens dans les emails
  • ✅ Gardez votre navigateur à jour
  • ✅ Utilisez un gestionnaire de mots de passe
  • ✅ Vérifiez les URLs avant de naviguer

Pour les développeurs :

  • ✅ Validez toutes les entrées utilisateur
  • ✅ Utilisez des requêtes préparées contre les injections SQL
  • ✅ Chiffrez les données sensibles
  • ✅ Implémentez HTTPS/TLS
  • ✅ Protégez contre le XSS et le CSRF
  • ✅ Utilisez des headers de sécurité HTTP
  • ✅ Faites des tests de sécurité régulièrement
  • ✅ Gardez vos dépendances à jour

🔍 Signes d'une page web malveillante

  • ❌ URL bizarre ou légèrement différente du site normal
  • ❌ Certificat SSL invalide
  • ❌ Vous êtes redirigé vers un autre site
  • ❌ Demandes inattendues de mots de passe
  • ❌ Beaucoup de publicités ou pop-ups
  • ❌ Fautes d'orthographe ou de grammaire
  • ❌ Le design est différent de la version normale
