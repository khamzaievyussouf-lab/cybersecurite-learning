🎯 Tests de Pénétration (Penetration Testing)

Qu'est-ce qu'un test de pénétration ?

Un test de pénétration (pen test) est une évaluation de sécurité autorisée où des professionnels tentent d'exploiter les vulnérabilités d'un système pour en identifier les failles.

Types de tests de pénétration

1️⃣ Black Box Testing

Le testeur n'a aucune information préalable sur le système.

  • Approche : Comme un attaquant réel
  • Avantage : Plus réaliste
  • Inconvénient : Plus long et coûteux

2️⃣ White Box Testing

Le testeur a accès complet aux informations (code source, architecture, etc.).

  • Approche : Inspection détaillée
  • Avantage : Identification complète des failles
  • Inconvénient : Moins réaliste

3️⃣ Gray Box Testing

Le testeur a des informations partielles (entre black et white box).

  • Approche : Équilibrée
  • Avantage : Bon compromis

📋 Phases d'un test de pénétration

Phase 1️⃣ : Planification

  • Définir les objectifs du test
  • Établir le périmètre
  • Fixer les dates et heures
  • Définir les risques acceptables
  • Obtenir l'autorisation par écrit

Phase 2️⃣ : Reconnaissance

  • Recherche OSINT (Open Source Intelligence)
  • Identification des domaines et sous-domaines
  • Recherche d'informations publiques
  • Identification des employés clés
  • Recherche sur les réseaux sociaux

Phase 3️⃣ : Scanning

  • Scan de ports avec Nmap
  • Identification des services actifs
  • Identification des versions logicielles
  • Scan de vulnérabilités
  • Construction d'une carte du réseau

Phase 4️⃣ : Énumération

  • Énumération des utilisateurs
  • Énumération des partages réseau
  • Énumération des services
  • Énumération des bases de données
  • Identification des points d'entrée potentiels

Phase 5️⃣ : Exploitation

  • Exploitation des vulnérabilités trouvées
  • Gain d'accès initial
  • Validation des failles
  • Documentation des exploitations
  • Évaluation de l'impact

Phase 6️⃣ : Post-exploitation

  • Reconnaissance interne
  • Escalade de privilèges
  • Maintien d'accès
  • Extraction de données
  • Couverture des traces (documentation)

Phase 7️⃣ : Rapport

  • Documentation des failles
  • Évaluation de la sévérité
  • Recommandations de correction
  • Preuves de concepts (PoC)
  • Résumé exécutif

🔍 Méthodologies reconnues

OWASP Testing Guide

Guide pour tester les applications web.

  • 13 catégories de tests
  • 100+ cas de test
  • Référence pour le web hacking

NIST SP 800-115

Méthodologie générale des tests de sécurité.

  • Phase 3 principaux : Planning, Execution, Reporting
  • Guide complet

PTES (Penetration Testing Execution Standard)

Standard professionnel avec 7 phases.

  • De la reconnaissance à la communication des résultats
  • Approche systématique

📊 Rapport de test de pénétration

Contenu recommandé :

  • Résumé exécutif : Vue d'ensemble pour les décideurs
  • Résultats détaillés : Chaque vulnérabilité trouvée
  • Preuves de concepts : Démonstration des failles
  • Recommandations : Comment corriger
  • Calendrier de correction : Priorités
  • Méthodologie : Comment le test a été effectué
  • Avertissements légaux : Cadre légal

⚠️ Niveaux de sévérité

Sévérité Description Délai de correction
🔴 Critique Accès système, données sensibles compromises Immédiatement
🟠 Haute Accès non autorisé, escalade de privilèges 1-2 semaines
🟡 Moyenne Fuite d'information, services dégradés 1 mois
🟢 Basse Impact limité, détection difficile À planifier

💼 Rôles importants

Testeur de pénétration

  • Effectue les tests
  • Identifie les failles
  • Crée les PoC
  • Rédige le rapport

Équipe rouge (Red Team)

  • Simule une attaque réelle
  • Peut utiliser l'ingénierie sociale
  • Tests plus complets et réalistes

Équipe bleu (Blue Team)

  • Défend le système
  • Répond aux attaques
  • Améliore la sécurité

✅ Bonnes pratiques

  • ✅ Toujours obtenir l'autorisation écrite
  • ✅ Respecter le périmètre défini
  • ✅ Documenter tout ce que vous faites
  • ✅ Signaler les failles responsablement
  • ✅ Ne pas causer de dommages intentionnels
  • ✅ Respecter la confidentialité des données
  • ✅ Tester en dehors des heures critiques

🎓 Certifications en Pen Testing

  • OSCP : Offensive Security Certified Professional (très difficile)
  • CEH : Certified Ethical Hacker
  • GPEN : GIAC Penetration Tester
  • GWAPT : GIAC Web Application Penetration Tester
  • eCPPT : eLearnSecurity Certified Professional Penetration Tester
